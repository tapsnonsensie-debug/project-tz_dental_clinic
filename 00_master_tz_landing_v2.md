# 00_master_tz_landing.md — Базовый стандарт коммерческого лендинга РФ

> Назначение: стандарт секций, legal-требований и placeholder-карты.
> Используется совместно с `claude_landing_workflow_master_v4.md`.
> Этот файл не содержит промптов и инструкций для агента.

## 1. Типовые секции лендинга

Порядок рекомендованный. Состав — по ТЗ проекта.

### 1.1 Header
**Цель:** ориентация, быстрый доступ к CTA.
- Логотип `[logo]`
- Навигация по якорям: только `href="#section"`, не scrollIntoView
- Телефон `[phone]` — кликабельный `tel:`
- Кнопка CTA (основное действие)
- Burger-меню на мобиле — реализовывать строго по шаблону из `WebDesignPrototypeSkill.md`:
  - slide справа, `position: fixed`, `width: min(320px, 85vw)`, `height: 100dvh`
  - отдельный overlay с `z-index` ниже меню
  - `body.menu-open` блокирует скролл страницы через `position: fixed; width: 100%`
  - закрытие: по overlay, по пункту меню, по Escape
  - `aria-expanded` на кнопке бургера, `aria-hidden` на меню

### 1.2 Hero
**Цель:** захват внимания, главный оффер, первичный CTA.
- H1 — главная ценность/оффер
- Подзаголовок — конкретизация: для кого, результат
- Первичный CTA (форма или модалка)
- Вторичный CTA (мессенджер или «узнать подробнее»)
- Визуал `[16:9 image]` или `[hero-bg]`
- Доверительный элемент: цифра или факт `[факт]`

### 1.3 Социальное доказательство — быстрая полоса
**Цель:** мгновенное доверие сразу после Hero.
- 3–5 логотипов клиентов `[client-logo]` ИЛИ 2–3 цифры `[цифра]` + подпись
- Не выдумывать. Только `[placeholder]`.

### 1.4 О компании / Преимущества
**Цель:** закрыть вопрос «почему вы».
- 3–6 преимуществ с иконками `[icon]` + описание 1–2 строки
- Опционально: фото команды `[team-photo]` или офиса `[office-photo]`

### 1.5 Услуги / Продукты
**Цель:** показать что предлагается, помочь выбрать.
- Список услуг/продуктов с описанием
- Цена или диапазон `[цена]`
- CTA под каждой позицией или общий CTA блока
- Табы или аккордеон если услуг > 4

### 1.6 Как это работает / Процесс
**Цель:** снять страх неизвестности.
- 3–5 шагов с нумерацией
- `[icon]` + заголовок + короткое описание
- Финальный результат последнего шага

### 1.7 Кейсы / Результаты
**Цель:** доказать результат на примерах.
- 2–4 кейса: задача → решение → результат
- `[case-image]`, `[client-name]`, `[результат]`
- Не выдумывать. Всё — плейсхолдеры до получения реального контента.

### 1.8 Отзывы
**Цель:** социальное доказательство.
- 3–6 отзывов: `[Имя Фамилия]`, `[должность, компания]`, `[текст отзыва]`, `[avatar]`
- Не выдумывать имена, тексты, компании.

### 1.9 FAQ
**Цель:** закрыть типовые возражения.
- 5–8 вопросов в аккордеоне (keyboard-accessible)
- Вопросы под нишу из ТЗ или `[Вопрос N]` / `[Ответ N]`

### 1.10 Форма / CTA-блок
**Цель:** получить заявку.
- Заголовок с конкретной ценностью (не «Оставьте заявку»)
- Поля: имя + телефон или email — минимум
- Кнопка с конкретным действием: «Записаться», «Получить расчёт»
- Consent checkbox — **unchecked по умолчанию**, блокирует submit
- Ссылка на политику конфиденциальности под формой
- Состояния: loading / success / error (`aria-live="polite"`)
- Защита от двойного submit
- Endpoint: `[FORM_ENDPOINT]` из `.env.example`

### 1.11 Footer
**Цель:** legal, контакты, навигация.
- Логотип `[logo]`
- Контакты: `[phone]`, `[email]`, `[address]`
- Ссылки: Политика конфиденциальности, Публичная оферта, Политика cookie
- Реквизиты: `[Название компании]`, ИНН `[ИНН]`, ОГРН `[ОГРН]`, `[Юридический адрес]`
- Copyright: © `[год]` `[Название компании]`
- **Кнопка «Наверх»** — обязательна на всех лендингах длиннее 3 экранов. Реализовывать по шаблону из `WebDesignPrototypeSkill.md`: `position:fixed`, правый нижний угол, появляется после 400px скролла, `z-index:900`

## 2. Опциональные секции

| Секция | Когда добавлять |
|---|---|
| Квиз / Калькулятор | Сложный продукт, цена зависит от параметров |
| Видео / Демо | Есть реальное видео `[video-embed]` |
| Команда | Услуги где важна экспертиза конкретных людей |
| Сертификаты / Лицензии | Медицина, юриспруденция, строительство |
| Партнёры | B2B, франшиза |
| Карта / Как добраться | Офлайн-бизнес |
| Тарифы | SaaS, подписки, курсы |

## 3. Legal-требования (152-ФЗ, frontend-слой)

> Frontend-слой не заменяет юридическую проверку. Тексты документов — юрист.

### 3.1 Обязательные элементы

- Consent checkbox под каждой формой — `checked=false` по умолчанию
- Текст: «Согласен с [политикой конфиденциальности]» — ссылка обязательна
- Submit заблокирован без согласия
- Cookie-баннер при первом визите (скрывается после принятия)
- Footer — legal-ссылки + реквизиты

### 3.2 Обязательные legal-страницы

| Файл | Содержание |
|---|---|
| `legal/privacy.html` | Политика конфиденциальности и обработки ПД |
| `legal/offer.html` | Публичная оферта |
| `legal/cookies.html` | Политика использования cookie |

Все три — структурные заглушки. Финальные тексты — заказчик или юрист.

### 3.3 Запреты

- Не заявлять соответствие 152-ФЗ без проверки юристом
- Не логировать согласие только на frontend
- Не скрывать checkbox и не делать его визуально неотличимым

## 4. Placeholder-карта

Все плейсхолдеры — вместо выдуманного контента. Заменяет заказчик перед публикацией.

### Визуалы
```
[logo]              — логотип компании
[hero-bg]           — фоновое изображение Hero
[16:9 image]        — изображение 16:9
[team-photo]        — фото команды
[office-photo]      — фото офиса/производства
[case-image]        — изображение кейса
[avatar]            — фото клиента для отзыва
[video-embed]       — embed-код видео
[map]               — embed-код карты
[icon]              — иконка
[client-logo]       — логотип клиента/партнёра
[certificate]       — скан сертификата/лицензии
```

### Тексты
```
[Название компании] — юридическое или торговое название
[Имя Фамилия]       — имя сотрудника или клиента
[должность]         — должность и компания
[текст отзыва]      — отзыв клиента
[Вопрос N]          — вопрос FAQ
[Ответ N]           — ответ FAQ
[client-name]       — компания-клиент в кейсе
[результат]         — цифра результата в кейсе
[факт]              — доверительный факт в Hero
[цена]              — цена или диапазон
[цифра]             — статистический показатель
```

### Контакты и реквизиты
```
[phone]             — номер телефона
[email]             — email
[address]           — физический адрес
[ИНН]               — ИНН компании
[ОГРН]              — ОГРН или ОГРНИП
[Юридический адрес] — юридический адрес
[год]               — текущий год (copyright)
```

### Интеграции
```
[FORM_ENDPOINT]     — endpoint формы (в .env.example)
[METRIKA_ID]        — ID Яндекс.Метрики (в .env.example)
[GA4_ID]            — Measurement ID GA4 (в .env.example)
[CRM_KEY]           — API-ключ CRM (в .env.example)
```
